// What is miru

Multimodal models are black boxes. Miru opens them.

Built on FastAPI, Miru is an explainability layer in front of any vision-language backend (CLIP shipped, LLaVA on the roadmap). It exposes a single REST surface for inference and produces a structured reasoning trace, an attention map, and an optional PNG overlay — for every request.

①

Pluggable backends, one interface

A VLMBackend registry mounts CLIP today and any future VLM tomorrow. The same FastAPI surface routes to whichever backend is requested — clients don't care which model produced the trace.

②

Server-Sent Events for live reasoning

POST /analyze/stream emits step events as the model produces each reasoning step, then a final trace event byte-equivalent to the synchronous endpoint. Keepalive comments hold the connection open behind any proxy.

③

Privacy-safe persistence

Set MIRU_RECORD=1 and every trace lands as JSONL. Raw image bytes never touch disk — only a SHA-256 hash. The overlay PNG is stripped before write. Local FS, S3, GCS, anything fsspec speaks.

④

Built for fine-tuning pipelines

Recorded JSONL is ready for downstream training. miru record export --format csv flattens to a fine-tunable schema in one command. Per-batch immutable files are object-storage-native.

// quickstart
# start the API
$ uvicorn miru.main:app

# single-shot inference
$ curl -X POST localhost:8000/analyze \
    -d '{"image_b64":"…","question":"?"}' \
    -H "Content-Type: application/json"

# streaming reasoning
$ curl -N localhost:8000/analyze/stream \
    -d '{"image_b64":"…","question":"?"}' \
    -H "Content-Type: application/json"

# persist every trace
$ MIRU_RECORD=1 uvicorn miru.main:app

$ miru record list
$ miru record export --out data.jsonl
// Capabilities

Everything you need, nothing you don't.

Five primitives. Production-grade. Zero abstractions you didn't ask for.

Image Analysis

Pluggable VLM backends behind a single VLMBackend interface. CLIP ViT shipped. Mock backend for tests. Lazy weight load on first call — no startup penalty.

SSE Streaming

Server-Sent Events deliver each reasoning step the moment it's available. Keepalive frames, configurable per-request timeout, clean error events on backend failure.

Dataset Recorder

Threaded JSONL writer. Per-batch immutable files for object-storage compatibility. Local, S3, GCS, memory — anything fsspec opens.

Privacy-safe Traces

Raw image bytes never persisted. SHA-256 hash only. Overlay PNG stripped before write. Errors in the recorder never propagate to the request path.

Overlay Support

Pass ?overlay=true and the response carries a base64 PNG with a bilinearly upsampled attention heatmap blended over the source image. Pure-NumPy + Pillow, no matplotlib.

Built for the CLI

miru record list and miru record export --format jsonl|csv turn recorded data into something a fine-tuning script can consume in one line.

// Architecture

Six stages. No magic.

An image and a question come in. A trace, a stream, and a JSONL line go out.

// Code

Two languages, both terse.

From the CLI or from your browser.

python
from miru import MockVLMBackend, ReasoningTracer
import numpy as np

backend = MockVLMBackend()
tracer  = ReasoningTracer()

image = np.zeros((8, 8, 3), dtype=np.float32)
out   = backend.infer(image, "What is shown?")

trace = tracer.trace(out, backend.name, latency_ms=0.0)
print(trace.answer)
print(trace.steps)            # [ReasoningStep(...), ...]
print(trace.attention_map)    # 16×16 normalized heatmap
javascript
const resp = await fetch("/analyze/stream", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ image_b64, question })
});

const reader = resp.body.getReader();
while (true) {
  const { value, done } = await reader.read();
  if (done) break;
  // SSE frame: parse event/data and render
  render(parseSSE(value));
}
